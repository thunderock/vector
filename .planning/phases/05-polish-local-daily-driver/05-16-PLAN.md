Four wgpu pipelines exist, compile, and pass geometry unit tests, but none are instantiated and none are invoked from the live render loop. UI-SPEC §11 specifies the exact draw order:
1. Per-pane compositor (grid + Phase-4 active-pane border) — already wired (app.rs:300–380).
2. Tint stripe (window chrome, above all panes).
3. OSC-8 hover underline (lives inside grid — already part of grid attributes).

6. Profile picker scrim + modal (modal, dims everything).

**HIGH-2 (05-REVIEWS.md) architectural decision LOCKED.** The four chrome pipelines do NOT live inside `RenderHost`. They live in a NEW struct `ChromePipelines` defined in `crates/vector-app/src/chrome.rs`, owned by `AppWindow` as a parallel field to `render_host`:

```rust
// crates/vector-app/src/app.rs — AppWindow struct (existing fields preserved, NEW field added)
pub(crate) struct AppWindow {
    pub window: Arc<Window>,
    pub render_host: Option<RenderHost>,
    pub chrome_pipelines: Option<ChromePipelines>,   // NEW — parallel to render_host (HIGH-2)

```

This means `app.windows[id].chrome_pipelines` and `app.windows[id].render_host` can be borrowed independently in `render_window` — the wgpu borrow conflict the reviewer flagged ("tint_mut() returning &mut self on RenderHost while other accessors take &self") goes away. `ChromePipelines::new(device, format)` is called immediately after `RenderHost::new(window)` succeeds, using the just-created device + format borrows. Construction is colocated in `resumed()`, `handle_new_tab()`, and `AppShortcut::SpawnNewWindow` (Plan 05-14).

                   );

               }

                   crate::toast::ToastMode::Action { .. } => u32::MAX,

               chrome.toast.draw(&mut rpass);

               let content_h = surface_h;

           }

   fn parse_hex_rgba(hex: &str) -> Option<[f32; 4]> {
       let s = hex.trim_start_matches('#');
       if s.len() != 6 { return None; }
       let r = u8::from_str_radix(&s[0..2], 16).ok()? as f32 / 255.0;
       let g = u8::from_str_radix(&s[2..4], 16).ok()? as f32 / 255.0;
       let b = u8::from_str_radix(&s[4..6], 16).ok()? as f32 / 255.0;

   }
