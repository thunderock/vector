Established main.rs pattern for sync-channel hand-off (lines 67, 209-213):
```rust
let (handle_tx, handle_rx) = std::sync::mpsc::sync_channel::<tokio::runtime::Handle>(1);
// ... inside io-thread tokio runtime:
let _ = handle_tx.send(rt.handle().clone());
// ... on main thread after App is built:
if let Ok(handle) = handle_rx.recv() {
    application.set_tokio_handle(handle);
} else {
    tracing::warn!("tokio handle not received from I/O thread; auth disabled");
}
```

The io-thread already has `Mux::install(Arc::clone(&mux));` (main.rs:88), so `Mux::try_get()` works for the actor's internal calls. We pass `Arc::clone(&mux)` explicitly into `DevTunnelsActor::new` because the actor's `mux` field is required.

`Handle::current()` is the canonical way to get the current runtime handle from within `rt.block_on(...)` — equivalent to `rt.handle().clone()` but available from inside the async block without capturing `rt`.
