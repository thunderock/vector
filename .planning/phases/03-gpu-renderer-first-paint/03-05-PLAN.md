  <behavior>
    - **Coalescing (D-47):** the I/O actor's reader branch no longer emits one `PtyOutput` per chunk. Instead it appends bytes into a shared `Arc<parking_lot::Mutex<BytesMut>>`. A new tokio task (`frame_tick`) wakes every 8 ms (or whenever a size threshold — e.g., 8 KiB — is hit; the actor task signals via a `tokio::sync::Notify` when the threshold is crossed) and drains the buffer, emitting ONE `UserEvent::PtyOutput(drained_bytes)` per drain.
    - **Empty buffer = no event:** if the coalesce buffer is empty on tick, no event fires. This keeps idle CPU near zero (RENDER-03).
    - **Render-on-dirty (D-44):** the App::user_event handler calls `window.request_redraw()` ONLY when (a) PtyOutput drained non-empty bytes, OR (b) input event (KeyboardInput / MouseInput / CursorMoved during drag / MouseWheel) fires. The redraw closure inside `WindowEvent::RedrawRequested` already gates on `Term::damage()` returning non-Full / non-empty Partial — if the compositor sees no damage and no input, it skips the GPU work. This is enforced by adding an early-return in `Compositor::render` if `damage() == TermDamage::Partial(empty)` AND selection is unchanged.
    - **LPM throttle (D-46):** `lpm.rs` spawns an observer task. **Primary path (per 03-RESEARCH.md Open Question caveat):** try the objc2-foundation block-observer API (`NSNotificationCenter::defaultCenter().addObserverForName_object_queue_usingBlock(NSProcessInfoPowerStateDidChangeNotification, ...)`); if the API spike fails (~30 min), fall back to a 1Hz polling task. Either way, the task sends `UserEvent::LpmChanged(now_enabled)` through the EventLoopProxy. App tracks the bool; when LPM is enabled, frame_tick's interval extends to 33 ms (~30 fps cap) instead of 8 ms (~120 fps Fifo cap). A `tracing::info!` fires on each transition.
    - **Resize debounce (D-49):** `WindowEvent::Resized` triggers `host.resize(...)` (cheap — surface.configure) immediately; the Term::resize / transport.resize side debounces 50 ms — store a `last_resize_at: Instant` on App and only fire the `input_bridge.send_resize` if 50 ms has elapsed since the last call, OR use a `tokio::time::sleep_until` task. Simpler approach for v1: store the pending (cols, rows) and only emit on RedrawRequested if `(now - last_resize_at) >= 50ms`. (Pure-Rust, no spawned task.)

         ```rust
         use std::time::Instant;
         use tokio::time::{interval, Duration};

         let mut iv = interval(Duration::from_millis(8));
         iv.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Skip);
         let mut last_drain_at = Instant::now();
         loop {
             tokio::select! {
                 _ = iv.tick() => {}
                 _ = coalesce.notify.notified() => {} // threshold crossed → wake early
             }
             // LPM gating: 30fps cap when enabled, 120fps otherwise (Fifo limits real rate).

                 Duration::from_millis(33)

                 Duration::from_millis(8)
             };
             if Instant::now().duration_since(last_drain_at) < period {
                 tokio::task::yield_now().await;
                 continue;
             }
             let bytes = coalesce.drain();
             last_drain_at = Instant::now();
             if !bytes.is_empty() {
                 if proxy.send_event(UserEvent::PtyOutput(bytes)).is_err() { return; }
             }
         }
         ```
