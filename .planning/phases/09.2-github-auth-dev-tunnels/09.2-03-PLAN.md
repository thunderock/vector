<objective>
Swap the app-side identity provider from Microsoft to GitHub and remove the app-side Microsoft auth surface (D-1). This is a near-mechanical rename driven by the exhaustive per-symbol change table in RESEARCH "App-Side Wiring Inventory" — every symbol, file, and line number is enumerated there. The layers below (`api.rs`, `AuthProvider::GitHub`, transport) are already provider-agnostic; this plan only changes the driver wiring and UI copy.

Note: Phase 9.1 (commits 5ffa51e, 622d3fe) just renamed these symbols GitHub→Microsoft. This plan reverses most of that cosmetic direction AND swaps the underlying driver to the real `GitHubAuth` from Plan 09.2-02.

Purpose: GitHub becomes the only app sign-in path; the menu/modal/footer all say GitHub; the app builds against the new driver.
Output: renamed app-side symbols, `microsoft_auth_modal.rs`→`github_auth_modal.rs`, GitHub-pointed `main.rs`, green `vector-app` build.
</objective>
