# Phase 09.2 Plan 03: App-Side GitHub Auth Swap (D-1) Summary

**Swapped vector-app's identity provider from Microsoft to GitHub end-to-end: `main.rs` now constructs `GitHubAuth::new(GITHUB_DEVTUNNELS_CLIENT_ID)` + `GitHubTokenStore::for_vector()`, the actor emits `AuthProvider::GitHub`, and every menu item / device-flow modal / in-picker button / footer / toast says GitHub. The whole workspace builds, is clippy-clean, and all tests pass — the compiling tree (broken since 09.2-02 deleted the Microsoft driver) is restored.**

## What Changed

Near-mechanical rename + copy-flip driven by RESEARCH "App-Side Wiring Inventory", in two atomic commits.

### Task 1 — driver + data wiring (`fc830b0`)
- `lib.rs` `UserEvent`: `Microsoft*` variants → `GitHub*` (fields unchanged); `DevTunnelsAuthRequired` doc reworded.
- `devtunnels_actor.rs`: import `auth::{GitHubAuth, GitHubAuthError, GitHubTokenStore, GitHubTokens}`; fields `github_auth`/`token_store: Arc<GitHubTokenStore>`; `Command::StartGitHubSignIn`/`SignOutGitHub`; `auth_provider()` + reconnect `auth_factory` build `AuthProvider::GitHub`; `handle_start_github_signin`; `save_then_announce` emits `GitHubSignedIn`; `GitHubAuthError` throughout.
- `main.rs`: `GitHubAuth::new(vector_tunnels::auth::GITHUB_DEVTUNNELS_CLIENT_ID)` + `GitHubTokenStore::for_vector()`.

### Task 2 — UI surface + copy (`00c4de6`)
- `menu.rs`: `install_github_menu_items`, `rebuild_github_signin_section`, `github_signin_menu_rows` ("Sign in/out … GitHub"), `GitHubMenuRefs`/`GITHUB_MENU_REFS`, ObjC `VectorGitHubMenuTarget`, selectors `githubSignIn:`/`githubSignOut:` (synced with `setAction(sel!(...))`), NSMenuItem titles.
- `microsoft_auth_modal.rs` → `github_auth_modal.rs` (`git mv`): `GitHubAuthDeviceFlowModal`/`Ctx`/`Responder`, panel title "Sign in with GitHub", ObjC `VectorGitHubAuthModalResponder`; `lib.rs` `pub mod github_auth_modal` (moved to alpha position).
- `devtunnels_modal.rs`: in-picker button "Sign in with GitHub", selector `githubSignIn:` → `UserEvent::GitHubSignInRequested`, `footer_copy(NotSignedIn)` → "Sign in with GitHub to list Dev Tunnels.", tint → `GITHUB_PURPLE_SRGB`, `SignedInOtherProvider` test fixture → Microsoft.
- `app.rs`: `github_auth_modal` field, `GitHub*` event arms (retargeting `Command::StartGitHubSignIn`/`SignOutGitHub`), `install_github_menu_items`/`rebuild_github_signin_section` calls, toasts ("Signed in to GitHub." / "GitHub sign-in cancelled." / "GitHub sign-in failed: …" / "Signed out of GitHub."), `NotSignedIn` footer toast, DevTunnel pane chrome stripe → `GITHUB_PURPLE`.
- tests: `microsoft_signin_menu.rs` → `github_signin_menu.rs` (assertions flipped to GitHub); `devtunnels_picker.rs` `NOT_SIGNED_IN` const flipped.

## Tint Decision

The Microsoft-blue tint (used in two places: the in-picker sign-in button `setContentTintColor` and the active DevTunnel pane chrome stripe) was re-pointed to **GitHub purple `#7a3aaf`** (`GITHUB_PURPLE` / `[0.478, 0.227, 0.686, 1.0]`). The constant already existed in `vector_render::tint_stripe` (a Phase-6 legacy left dormant after the Codespaces revert), so no new color needed resolving. This is cosmetic and outside the success-criteria grep scope, but re-pointing keeps the brand consistent now that GitHub is the only provider.
