## Scope discipline

## Gaps

### Gap A — Local pane exit handler (Phase 4-05 stub)

- **Symptom:** Typing `exit` in a local pane leaves the prompt on screen; pane never closes.
- **Root location:** `crates/vector-app/src/app.rs:1872-1874` — `UserEvent::PaneExited` arm is `tracing::info!(?pane_id, "pane exited (Plan 04-05 will render sentinel)")` only. No call into `Mux::close_pane` and no sentinel render.
- **Producer is correct:** `crates/vector-app/src/pty_actor.rs:232` emits `UserEvent::PaneExited(pane_id)` on PTY EOF.
- **Acceptance:** `exit` in a local shell closes the pane (or renders a clearly-marked `[exited]` sentinel dismissable with Cmd-W). No regression to remote-pane behavior (remote panes go through `Reconnecting` instead of `PaneExited`).

### Gap B — Codespaces UI cleanup (Phase 7 pivot)

- **Symptom:** Vector menu still advertises `Codespaces…` (Cmd-Shift-G); the entire Codespaces UI stack is wired in despite the 2026-05-19 pivot.
- **Surviving code (incomplete inventory):**
  - `crates/vector-codespaces/` — full crate
  - `crates/vector-app/src/codespaces_actor.rs` — `spawn_fetch_codespaces`, `spawn_poll_row`, `spawn_start_codespace`, `build_client_from_keychain`, `has_keychain_token`
  - `crates/vector-app/src/codespaces_modal.rs` — `CodespacesPickerModal`
  - `crates/vector-app/src/lib.rs:19-20,108-117` — `pub mod codespaces_actor` + `pub mod codespaces_modal` + `OpenCodespacesPicker` / `CodespacesLoaded` / `CodespacesLoadFailed` / `CodespaceStateChanged` UserEvent variants
  - `crates/vector-app/src/menu.rs:436-452` — `Codespaces…` menu item, `openCodespaces:` selector in `install_phase6_items`
  - `crates/vector-app/src/app.rs` — `codespaces_modal` + `codespaces_client` App fields, `handle_open_codespaces_picker`, `handle_codespaces_loaded`, related event arms
  - `crates/vector-app/src/relative_time.rs` — `state_label` / `state_color` over `vector_codespaces::CodespaceState`
  - `crates/vector-app/src/auth_actor.rs:18` — `use vector_codespaces::{AuthError, GitHubAuth, TokenStore}` (shared with non-codespaces auth)
  - `crates/vector-arch-tests/tests/no_token_in_debug_or_log.rs:11,131` — scan paths reference `vector-codespaces/src`
- **Acceptance:** No `Codespaces…` menu item. No `codespaces_modal` / `codespaces_actor` module. No Codespaces UserEvent variants. The `vector-codespaces` crate is either deleted or split (keep token-store / GitHubAuth under a new name like `vector-github-auth` if `auth_actor` still depends on it; delete the rest). Arch-tests still pass; release build still compiles; GitHub OAuth (if still desired anywhere) still works.

### Gap C — Microsoft menu items never installed (Phase 8-05 wiring)

- **Symptom:** No `Sign in with Microsoft` menu item visible in the running app, so users can't reach Microsoft device-flow at all.
- **Root cause:** `crates/vector-app/src/menu.rs:563` defines `install_microsoft_menu_items(...)` but `grep -rn install_microsoft_menu_items crates/` shows ZERO call sites anywhere in the workspace. Phase 8-05 SUMMARY claims this ships but the install was never wired into the app startup path.
- **Acceptance:** `install_microsoft_menu_items` is called at startup (likely in `main.rs` after `install_main_menu` and `install_phase6_items` complete). The menu items appear in the Vector submenu. `MicrosoftMenuTarget` selectors route through the EventLoopProxy to fire `MicrosoftSignInRequested` / `MicrosoftSignOutRequested` / `OpenDevTunnelsPickerMenu`. `rebuild_microsoft_signin_section` toggles the visible row based on token presence.

### Gap D — Picker has no in-modal sign-in affordance (Phase 8-05 UX gap)

- **Symptom:** `Cmd-Shift-T` opens the Dev Tunnels picker. If not signed in, the picker shows footer text `Sign in with GitHub or Microsoft to list Dev Tunnels.` but offers no clickable button or auto-redirect. User is stranded.
- **Choice point (decide during discuss-phase):**
  1. Add a `Sign in with Microsoft` button to the `NotSignedIn` state of `DevTunnelsPickerModal` (active style; calls into the same actor command as the menu item).
  2. Auto-present the `MicrosoftAuthDeviceFlowModal` on top of the picker when `FooterState::NotSignedIn` is computed.
  3. Both — button visible + a one-time auto-present on first open.
- **Acceptance:** From `Cmd-Shift-T` with no Microsoft token in the keychain, the user can reach a successful Microsoft sign-in without leaving the picker or hunting through menus.

### Gap E — Bundled `Vector.app` has no icon (Phase 1 / 10 distribution polish)

- **Symptom:** `Vector.app` dock + Finder + Cmd-Tab icon is a blank/black placeholder, not a Vector glyph.
- **Likely root cause:** `cargo-bundle` Info.plist `CFBundleIconFile` is unset, or no `.icns` is being produced/packaged. The `iconutil` step from the Phase 1 stack additions may not be wired into `xtask dmg`.
- **Acceptance:** Bundled `Vector.app` shows a Vector-branded icon in dock, Finder, and Cmd-Tab. CI-produced DMG includes it. Local `cargo xtask dmg` reproduces it on Apple Silicon.
