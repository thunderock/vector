## Implementation Decisions

### Gap A — Local pane exit handler (Phase 4-05 stub)

- **D-01:** `UserEvent::PaneExited(pane_id)` handler at `crates/vector-app/src/app.rs:1872-1874` calls into `Mux::close_pane(pane_id)` and removes the pane immediately. **No `[exited]` sentinel.** Tab closes when last pane closes; window closes when last tab closes. Matches iTerm / Apple Terminal default.
- **D-02:** **No config knob** for hold-on-exit in v1. Keeps scope tight. Reconsider in v2 if users ask.
- **D-03:** Remote panes (DevTunnel) are unaffected — they go through `Reconnecting` on EOF, never `PaneExited`. The handler change is local-pane only by virtue of where `PaneExited` is emitted (`pty_actor.rs:232` only fires it on PTY EOF, which `LocalDomain::reconnect_one_shot` maps to terminal closure via `Ok(None)`).

### Gap B — Codespaces decommission (Phase 7 pivot cleanup)

- **D-04:** **Delete the `vector-codespaces` crate wholesale.** Remove the crate directory, remove from workspace `Cargo.toml` members, remove all `vector_codespaces::*` imports across the workspace.
- **D-05:** **No GitHub OAuth in v1.** The only sign-in path is Microsoft (for Dev Tunnels). `crates/vector-app/src/auth_actor.rs` either deletes the GitHub-related code paths entirely or the file is removed if nothing else lives in it. `app.rs` event handlers that referenced `GitHubAuthStateChanged` / `OpenCodespacesPicker` / `CodespacesLoaded` / `CodespacesLoadFailed` / `CodespaceStateChanged` are removed.
- **D-06:** Codespaces UI surface removal inventory (researcher to verify completeness, planner to slice into a single coherent commit):
  - `crates/vector-codespaces/` — delete crate
  - `crates/vector-app/src/codespaces_actor.rs` — delete file
  - `crates/vector-app/src/codespaces_modal.rs` — delete file
  - `crates/vector-app/src/relative_time.rs` — delete file IF its only use is `CodespaceState`; otherwise strip the `state_label`/`state_color` functions
  - `crates/vector-app/src/lib.rs:19-20,108-117` — remove `pub mod codespaces_actor`, `pub mod codespaces_modal`, and the 4 codespaces UserEvent variants
  - `crates/vector-app/src/menu.rs:436-452` — remove the `Codespaces…` menu item and `openCodespaces:` selector from `install_phase6_items`. (May warrant renaming `install_phase6_items` if everything Phase-6-specific leaves it.)
  - `crates/vector-app/src/app.rs` — remove `codespaces_modal` + `codespaces_client` App fields, `handle_open_codespaces_picker`, `handle_codespaces_loaded`, all related event arms, and the 7 `vector_codespaces::*` references (lines 158, 162, 502, 1763, 1945, 2045 currently)
  - `crates/vector-arch-tests/tests/no_token_in_debug_or_log.rs:11,131` — remove `vector-codespaces/src` from `SCAN_PATHS`
- **D-07:** Acceptance check: `grep -rIn "codespace\|Codespace\|CODESPACE" crates/` returns only matches inside comments referencing the historical pivot (or zero matches), `cargo build --workspace` and `cargo test --workspace` both green, `cargo clippy --workspace --all-targets -- -D warnings` clean.
- **D-08:** This is the largest plan — handle as a single commit (or two: one for crate removal + workspace, one for app.rs/menu.rs surgery). The arch-test scan-path change must land in the same commit as the crate deletion or the lint will fail on the first commit.

### Gap C — Microsoft menu items wiring (Phase 8-05)

- **D-09:** Call `install_microsoft_menu_items` at app startup from `crates/vector-app/src/main.rs`, after the main menu and `install_phase6_items` complete (or whatever its renamed successor is per D-06).
- **D-10:** No new UserEvent variants needed — `MicrosoftSignInRequested` / `MicrosoftSignOutRequested` / `OpenDevTunnelsPickerMenu` already exist and are dispatched in `app.rs` (Plan 08-05 SUMMARY confirms).
- **D-11:** `rebuild_microsoft_signin_section` already exists at `menu.rs:638`; downstream code in app.rs that decides when to call it (after `MicrosoftSignedIn` / sign-out events) is already wired per Plan 08-05 SUMMARY. ### Gap D — Picker sign-in affordance (Phase 8-05 UX gap)

- **D-12:** Add a clickable `Sign in with Microsoft` button to `DevTunnelsPickerModal`. Visible only when `FooterState::NotSignedIn`. Uses `MICROSOFT_BLUE = [0.0, 0.471, 0.831, 1.0]` (UI-SPEC §Color — already reserved for active DevTunnel panes; reusing here is consistent).
- **D-13:** Click handler routes through the actor command channel (`DevTunnelsActor::Command::StartMicrosoftSignIn`) — same path the menu item fires. Reuses the existing `MicrosoftAuthDeviceFlowModal` presentation flow. Picker stays open behind the device-flow modal; on `MicrosoftSignedIn`, the picker auto-refreshes by re-firing `Command::Load`.
- **D-14:** **No auto-present.** Picker open in `NotSignedIn` state must be a deliberate user action followed by another deliberate action (clicking the button). No hijack-feel.
- **D-15:** **UI-SPEC amendment:** `FooterState::NotSignedIn` copy changes from `Sign in with GitHub or Microsoft to list Dev Tunnels.` to `Sign in with Microsoft to list Dev Tunnels.` (drops "GitHub or" — consistent with Gap B's deletion of GitHub auth). Update `footer_copy()` at `devtunnels_modal.rs:54-56` AND the dispatch branch in `app.rs:2160`. The character-for-character UI-SPEC tests will need to update with this change.

### Gap E — App icon (Phase 1 / 10 distribution polish)

- **D-16:** **Typographic V mark.** A simple monospaced "V" glyph on a dark rounded-square background.
- **D-17:** Generation pipeline: add an `xtask icon` subcommand (or extend the existing `xtask dmg` flow) that produces a 1024×1024 PNG via a small script — `cairo`/`resvg` rendering of an SVG template is fine, or pure-Rust with `tiny-skia` to avoid adding system deps. Then `iconutil --convert icns` produces `Vector.icns`. Result lives at `resources/Vector.icns` (or wherever cargo-bundle expects it).
- **D-18:** Wire `CFBundleIconFile = "Vector"` into `cargo-bundle` config (Cargo.toml `[package.metadata.bundle]`). Confirm `cargo xtask dmg` packages the icon into `Vector.app/Contents/Resources/`.
- **D-19:** Visual specifics — Claude's Discretion. Reasonable defaults: dark background (`#1a1a1a` or similar — close to ChromePalette dark surface), white/off-white V glyph, 18% corner radius matching macOS Big Sur+ icon grid. No gradient, no shadow. Can be replaced wholesale in v2 by dropping a new SVG/PNG into the same path.
