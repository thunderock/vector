## Decisions Made
- **Reuse the existing `UserEvent::MicrosoftSignInRequested`.** The arm in `app.rs:1805` already routes to `Command::StartMicrosoftSignIn`, so the button needed no new wiring path — the menu item and the picker button now converge on the same handler.
- **`setContentTintColor` over `NSAttributedString` tinting.** Picked the simpler, supported macOS 10.14+ API for the Microsoft-blue accent; the attributed-string title approach would have required `NSDictionary` + `NSForegroundColorAttributeName` plumbing for one button.
- **`NSBezelStyle::Push` over `NSBezelStyle::Rounded`.** `Rounded` is deprecated in current `objc2_app_kit`; `Push` matches `microsoft_auth_modal.rs::make_button` and avoids the deprecation warning under `-D warnings`.
- **Frame placement: centered horizontally inside the rows container.** Button sits at `((PANEL_W - 200)/2, ROWS_Y + (ROWS_H - 28)/2)`. The NotSignedIn state has no rows, so the empty rows area is the natural home; this also keeps the footer label at its locked frame untouched.
- **Auto-refresh gated by `devtunnels_modal.is_some()`.** Avoids firing `Command::Load` when the picker isn't even open (e.g., user signed in from the menu while no picker was visible).
