# Phase 09.1 Plan 03: Wire install_microsoft_menu_items at Startup Summary

Calls Plan 08-05's previously-orphaned `install_microsoft_menu_items` from `App::resumed`, surfacing "Sign in with Microsoft", "Sign out of Microsoft", and "Dev Tunnels…" in the Vector menu and restoring the only documented route into the device-flow modal aside from Cmd-Shift-T.

## What Shipped

- **crates/vector-app/src/app.rs (`App::resumed`):** added a guarded `unsafe { menu::install_microsoft_menu_items(mtm, proxy) }` call immediately after `menu::install_main_menu()`. The block obtains `MainThreadMarker::new()` (safe because winit guarantees `resumed` on the macOS main thread) and clones `self.proxy`. On failure to obtain either, the install is skipped with a `tracing::warn!` rather than panicking — matches the existing "missing proxy" softness in this file.
- **Comment update:** removed the stale "installed by main.rs" placeholder left by Plan 09.1-02 and replaced it with the actual Plan 09.1-03 / Gap C reference.
